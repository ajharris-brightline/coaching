Why We're Building This
One unified system for coaching and clinical care

We're building a unified care planning system for Brightline's coaching and clinical teams. Each team has a specific set of needs we'll be addressing — unification of care systems is the end goal.

This means a single platform where coaches and therapists build modular, evidence-based care plans, track measurement-based progress session by session, and give families real visibility into their child's care journey — powered by AI-assisted pre and post-session summaries.

This guide gets everyone on the project oriented with the same tools, vocabulary, and environment so we can build together.

Step 1 — Claude.ai

Set up your Claude account

Start here before anything else. Claude.ai is your conversational AI partner for thinking, learning, and iterating. Get comfortable with it before you open a terminal.

1
Create your account
Go to claude.ai and sign up with your Brightline work email. This makes it easy to link to the organization account later.
Go to Claude.ai ↗
2
Have your first real conversation
Don't treat Claude like a search engine — treat it like a knowledgeable colleague. Give context, ask follow-ups, push back when something's off. Try this as a starting point:
You
I'm joining a project to build a unified care planning system for a pediatric behavioral health company. We're using React, Claude Code, and GitHub. I'm not an engineer but I want to understand how everything fits together. Where should I start?
C
Great starting point. Let me break this into three layers: what you'll see (the React front end), how data moves (the API), and how everything gets published (GitHub + Netlify). Understanding these three things will let you follow along and contribute meaningfully without needing to write code from scratch...
3
Download the desktop app
The Claude desktop app lets you use Claude alongside any other app — great for referencing it while in VS Code or Figma without switching browser tabs.
Download Claude Desktop ↗
4
Four principles for great Claude conversations
Give context upfront. Tell Claude who you are, what project you're on, and what you already know. The more it understands your situation, the more useful it is.

Treat it as a thinking partner. Don't just ask for answers — ask Claude to help you think through problems, explain things simply, or challenge your assumptions.

Paste the actual material. If you want help with a document, a design, or some code — paste it in. Claude works best with the real thing in front of it.

Push back when it's wrong. Claude adjusts quickly. If a response misses the mark, say so directly and it will recalibrate.
Step 2 — Claude Code

Install Claude Code in Terminal

Claude Code is Claude inside your Terminal. It can read, write, and run files on your computer — this is how we actually build the React app. Follow these steps in order.

1
Open Terminal
On Mac: press ⌘ + Space, type Terminal, press Enter. On Windows: search PowerShell. This is your command line — you'll only ever need a handful of commands.
2
Check Node.js is installed
Paste this and press Enter. You should see something like v20.x.x. If you see "command not found", install Node from nodejs.org first.
node --version # Expected: v18.x.x or higher
3
Install Claude Code globally
Run this once. It installs Claude Code on your machine so you can use it from any project folder going forward.
npm install -g @anthropic-ai/claude-code # If you see a permissions error, add sudo: sudo npm install -g @anthropic-ai/claude-code
4
Navigate to your project folder
cd means "change directory" — it moves you into a folder. You need to be inside the project folder before launching Claude Code.
# Navigate to your Desktop, then into the project cd ~/Desktop cd brightline-clinical
5
Launch Claude Code
Type claude from inside your project folder and press Enter. A prompt appears — type your instruction exactly as you would in Claude.ai.
claude # Claude Code starts up. Type a task and press Enter: # "Read my App.jsx and explain what each component does" # "Add a loading spinner to the PreSession tab"
💡 Key difference from Claude.ai
Claude.ai is conversational — you talk, it responds. Claude Code is agentic — it reads your actual files, writes real code, and runs commands. Give it specific tasks, not questions.
6
When the session limit fills up
Context limits are normal. Your files are always safe — everything written to disk is preserved. Start a new session with claude and paste this to restore context instantly:
# Paste this at the start of every new Claude Code session: "I'm continuing work on the Brightline Unified Platform React app. The project is in this folder. Read the existing files in src/ and continue from where we left off."
Step 3 — GitHub

Set up GitHub

GitHub is version control, collaboration, and deployment in one. Think Google Drive for code — except every change is tracked forever and anyone on the team can see and build on it.

1
Create a GitHub account
Sign up at github.com with your work email. If you have one, just sign in.
Sign up for GitHub ↗
2
Create a new repository
A repository is a project folder on GitHub. Click New → name it brightline-clinical → set to Public → click Create Repository.
3
Push your project code to GitHub
From inside your project folder in Terminal, run these commands once. Replace the URL with your actual GitHub repo URL.
git init git add . git commit -m "Initial commit" git branch -M main git remote add origin https://github.com/yourusername/brightline-clinical.git git push -u origin main
4
Publish via GitHub Pages (for HTML files)
To share a public link from a single HTML file: Settings → Pages → Source: main branch → Save. The file must be named index.html. Goes live at yourusername.github.io/brightline-clinical
5
Connect Netlify for auto-deployment
Go to netlify.com → Add new site → Import from Git → GitHub. Select your repo. Every push to main now auto-deploys. Both Netlify and GitHub Pages URLs stay in sync.
Connect on Netlify ↗
Concepts — React & the API

How the build actually works

You don't need to memorize this — but understanding these two concepts will help you follow the build, have smarter conversations with Claude Code, and contribute meaningfully.

React + Vite — the front end

React is a way of building UIs out of reusable components — like Lego bricks. Vite compiles those components into a real webpage. The end result is plain HTML, CSS, and JavaScript that any browser can open.

PreSession.jsx
// A React component is just a function // that returns what to show on screen function PreSession({ patient }) { return ( <div className="pre-session"> <h2>Session Summary</h2> <p>Patient: {patient.name}</p> <p>GAD-7: {patient.gad7}</p> </div> ) } export default PreSession
🧩
Components are building blocks
PreSession, CarePlan, PostSession — each is its own file. They assemble into the full app in App.jsx. Claude Code creates and manages these files when you give it tasks.
📦
Props pass data in
{ patient } is a prop — data passed from a parent component. Right now it's fake placeholder data. Later, real EHR data comes in through the API.
🔄
State tracks what changes
When a provider switches between Provider View and Patient View — that's state changing. React automatically re-renders the right components when state updates.
⚡
npm run dev — your local server
Starts a local preview at localhost:5173. Looks like a live website — but it's only on your machine. Run npm run build to make it publishable.
The API — connecting to real data

The API is the messenger between what providers see on screen and where data actually lives. Think of it as the waiter between the restaurant (front end) and the kitchen (database).

Where we are — Phase 1
The React app has hardcoded placeholder data — Maya Chen's scores, session number, goals. We're building the UI and defining what shape real data should take.
What engineering adds — Phase 2+
Engineering builds API endpoints returning real data in the same shape. We swap hardcoded values for API calls. The UI doesn't change — only the data source does.
